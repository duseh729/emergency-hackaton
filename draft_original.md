# 기획서 초안 — 긴급 인수인계 해커톤 플랫폼

---

# 1. 서비스 개요

| 항목          | 내용                                                                                                                                       |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **서비스명**  | Daker — 긴급 인수인계 해커톤 플랫폼                                                                                                        |
| **한줄 소개** | 기능 명세서만 남기고 사라진 개발자의 문서를 기반으로, 바이브 코딩으로 웹서비스를 구현·배포하는 해커톤 운영 플랫폼                          |
| **개발 목표** | 해커톤 목록 조회 → 팀 구성(/camp) → 제출(Submit) → 리더보드까지 이어지는 전체 참가 흐름을 localStorage 기반으로 구현하고 Vercel에 배포한다 |

**타겟 사용자**

- 해커톤에 참가하거나 관심 있는 개발자/디자이너/기획자
- 팀원을 구하고 싶은 솔로 참가자

**기대효과**

- 해커톤 참가의 진입장벽 낮춤 (팀 구성 ~ 제출까지 한 플랫폼에서 해결)
- 공개 리더보드로 경쟁 동기 부여 및 투명한 평가 가시화

# 2. 페이지 구성

```
메인페이지 (/)
  ├── 해커톤 목록 (/hackathons)
  │     └── 해커톤 상세 (/hackathons/:slug)
  │           ├── 개요(Overview) / 안내(Info)
  │           ├── 평가(Eval)
  │           ├── 일정(Schedule)
  │           ├── 상금(Prize)
  │           ├── 팀(Teams)  ←→  팀원 모집 (/camp)
  │           ├── 제출(Submit)
  │           └── 리더보드(Leaderboard)
  └── 랭킹 (/rankings)
```

| 페이지      | URL                 | 주요 내용                                              | 주요 액션                                                    |
| ----------- | ------------------- | ------------------------------------------------------ | ------------------------------------------------------------ |
| 메인        | `/`                 | 서비스 소개 + 빠른 진입 카드 3개                       | 해커톤 보기 / 팀 찾기 / 랭킹 보기 버튼 클릭 → 각 페이지 이동 |
| 해커톤 목록 | `/hackathons`       | 해커톤 카드 리스트 (제목·상태·태그·기간·참가자수 표시) | 카드 클릭 → `/hackathons/:slug` 이동, (옵션) 상태·태그 필터  |
| 해커톤 상세 | `/hackathons/:slug` | 탭 구조로 7개 섹션 표시 (하단 상세 참고)               | 탭 전환, 팀 구성 초대, 제출 폼 입력·저장, 리더보드 확인      |
| 팀원 모집   | `/camp`             | 전체 팀 리스트 (쿼리 필터: `?hackathon=slug`)          | 팀 카드 보기, 팀 모집글 생성 폼 제출                         |
| 랭킹        | `/rankings`         | 글로벌 랭킹 테이블 (rank/닉네임/points)                | (옵션) 기간 필터: 최근 7일 / 30일 / 전체                     |

**해커톤 상세 (/hackathons/:slug) — 7개 섹션 구성**

| #   | 섹션                          | 표시 내용                                                                               | 주요 액션                                                                                            |
| --- | ----------------------------- | --------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- |
| 1   | 개요 / 안내 (Overview / Info) | 대회 한줄 요약, 팀 정책(솔로 허용·최대 인원), 공지사항, 규정·FAQ 링크                   | 규정/FAQ 링크 클릭                                                                                   |
| 2   | 평가 (Eval)                   | 평가 지표명, 평가 설명, 점수 가중치(참가자 30% / 심사위원 70%)                          | -                                                                                                    |
| 3   | 일정 (Schedule)               | 마일스톤 타임라인 목록 (이름·날짜·시간, 시간대 표시)                                    | -                                                                                                    |
| 4   | 상금 (Prize)                  | 순위별 상금 목록 (1st·2nd·3rd 등)                                                       | -                                                                                                    |
| 5   | 팀 (Teams)                    | 해당 해커톤 팀 카드 목록; 팀명·소개·모집 포지션·연락처 표시                             | "이 해커톤 팀 구성" 버튼 → `/camp?hackathon=slug`; 초대/수락/거절 버튼; (옵션) 팀 구성 유의사항 탭   |
| 6   | 제출 (Submit)                 | 제출 가이드, 제출 폼                                                                    | 메모(notes, 선택) + 파일(zip·pdf·csv 등, 해커톤 설정에 따라) 입력 후 "제출" 클릭 → localStorage 저장 |
| 7   | 리더보드 (Leaderboard)        | 순위 테이블 (rank·팀명·score·제출일); 점수는 해커톤 설정에 따라 계산(투표 or 평가 산식) | 점수 확인; 미제출 팀은 "미제출" 배지 표시·순위 제외                                                  |

**공통 UI**

- 상단 네비게이션 바: 메인 / `/hackathons` / `/camp` / `/rankings` 항목 고정 표시
- 로딩 스피너, 데이터 없음 안내, 에러 메시지 3종 상태 처리

# 3. 시스템 구성

```
[브라우저]
  ├── Next.js (App Router)
  │     ├── 파일 기반 라우팅 (app/ 디렉토리)
  │     └── 클라이언트 컴포넌트 레이어 ("use client" 위주)
  │           ├── app/page.tsx (메인)
  │           ├── app/hackathons/page.tsx (해커톤 목록)
  │           ├── app/hackathons/[slug]/page.tsx (해커톤 상세)
  │           ├── app/camp/page.tsx (팀원 모집)
  │           └── app/rankings/page.tsx (글로벌 랭킹)
  │
  └── localStorage (영속 데이터 저장소)
        ├── hackathons   — 해커톤 목록·상세 (초기값: 예시 JSON 시딩)
        ├── teams(camp)  — 팀 모집글
        ├── submissions  — 제출 내역 (plan/web/pdf)
        └── leaderboards — 리더보드 점수

[배포] Vercel (Next.js 최적화 배포, 외부 접속 가능)
[외부 API/DB 없음] — 별도 서버 없이 LocalStorage 전용 구동 (필요 시 Next.js Route Handlers로 확장 가능)
```

**데이터 초기화 전략**

- 앱 최초 실행 시 클라이언트에서 `예시자료/*.json`을 localStorage에 시딩
- 이후 CRUD는 localStorage에서 직접 처리 (새로고침 유지)

# 4. 핵심 기능 명세

---

## 4-1. 메인 페이지 (/)

서비스 진입점으로, 해커톤·팀 모집·랭킹으로 빠르게 이동할 수 있는 카드 3개를 제공합니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 페이지 진입 | 서비스 소개 및 진입 카드 3개 렌더링 | "해커톤 보기" → `/hackathons`, "팀 찾기" → `/camp`, "랭킹 보기" → `/rankings` 카드 버튼 표시 |

---

## 4-2. 해커톤 목록 (/hackathons)

등록된 해커톤을 카드로 나열하며, 상태·태그 필터로 원하는 대회를 빠르게 탐색할 수 있습니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 페이지 진입; (옵션) 상태 필터 선택(진행중·종료·예정), 태그 필터 선택 | localStorage `hackathons` 읽어 카드 렌더링; 필터 조건에 맞는 항목만 표시 | 제목·상태 배지·태그·시작일·종료일·참가자 수가 표시된 카드 목록; 카드 클릭 시 `/hackathons/:slug` 이동 |

---

## 4-3. 해커톤 상세 (/hackathons/:slug)

해커톤의 모든 정보를 7개 탭으로 제공하며, 팀 구성·제출·리더보드 확인까지 한 페이지에서 처리합니다.

### 4-3-1. 개요 / 안내 탭 (Overview / Info)

대회의 기본 정보와 참가 조건, 공지사항, 규정 링크를 안내합니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 탭 클릭 | 해당 slug의 overview·info 섹션 데이터 로드 | 대회 한줄 요약·팀 정책(솔로 허용 여부·최대 인원)·공지사항·규정·FAQ 링크 표시 |

### 4-3-2. 평가 탭 (Eval)

사용하는 평가 지표와 점수 산정 가중치를 보여줍니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 탭 클릭 | eval 섹션 데이터 렌더링 | 평가 지표명·평가 설명·점수 가중치(참가자 30% / 심사위원 70%) 표시 |

### 4-3-3. 일정 탭 (Schedule)

접수부터 결과 발표까지 주요 마일스톤을 타임라인으로 표시합니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 탭 클릭 | schedule 섹션 데이터 렌더링 | 마일스톤 타임라인 목록 표시 (이름·날짜·시간, 시간대 표시) |

### 4-3-4. 상금 탭 (Prize)

순위별 상금 목록을 보여줍니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 탭 클릭 | prize 섹션 데이터 렌더링 | 순위별 상금 목록 표시 (1st·2nd·3rd 등) |

### 4-3-5. 팀 탭 (Teams)

해당 해커톤에 참가 중인 팀 목록을 확인하고, 팀 구성·초대/수락/거절 액션을 처리합니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 탭 클릭; "이 해커톤 팀 구성" 버튼 클릭; 초대/수락/거절 버튼 클릭 | `/camp?hackathon=slug` 쿼리로 팀 목록 로드; 버튼 클릭 시 해당 액션 처리 또는 `/camp`로 이동 | 해당 해커톤 팀 카드 목록 (팀명·소개·모집 포지션·연락처); "이 해커톤 팀 구성" 버튼; 초대/수락/거절 버튼; (옵션) 팀 구성 유의사항 탭 |

### 4-3-6. 제출 탭 (Submit)

제출 가이드와 단계별 폼을 제공하며, 완료 즉시 리더보드에 반영됩니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 제출 폼 입력 (메모 optional, 파일 URL 등); "제출" 버튼 클릭 | 제출 가이드 표시 → 폼 검증(필수 항목 확인) → localStorage `submissions` 저장 | 제출 완료 알림; 리더보드 해당 팀 상태 "미제출" → "제출 완료" 반영 |

**제출 폼 입력 항목**

| 항목 | 형식 | 필수 여부 |
| ---- | ---- | --------- |
| 메모 (notes) | 텍스트 | 선택 |
| 파일 | zip·pdf·csv 등 (해커톤 설정에 따라 허용 형식 다름) | 해커톤 설정에 따라 |
| 기획서 (이번 해커톤) | text 또는 URL | 1차 제출 |
| 최종 웹링크 (이번 해커톤) | URL | 2차 제출 |
| 솔루션 PDF (이번 해커톤) | PDF URL | 3차 제출 |

### 4-3-7. 리더보드 탭 (Leaderboard)

해커톤 설정에 따른 점수 산식으로 순위를 표시하며, 미제출 팀은 순위에서 제외됩니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 탭 클릭 | `leaderboards[slug]` 읽기 + 제출 내역 join; 점수는 해커톤 설정에 따라 계산(투표 결과 or 평가 산식) | 순위 테이블 (rank·팀명·score·제출일); 미제출 팀은 "미제출" 배지 + 순위 제외 |

---

## 4-4. 팀원 모집 (/camp)

### 4-4-1. 팀 목록 보기

쿼리 파라미터 유무에 따라 전체 또는 특정 해커톤 팀만 필터링하여 보여줍니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 페이지 직접 진입(`/camp`) 또는 쿼리 진입(`/camp?hackathon=slug`) | 쿼리 있으면 해당 해커톤 팀만 필터; 없으면 전체 팀; localStorage `teams` 읽어 렌더링 | 팀 카드 목록 (팀명·모집 상태 배지·소개·포지션 태그·연락하기 버튼); (옵션) 채팅/쪽지 버튼 |

### 4-4-2. 팀 모집글 생성

팀명·소개·모집 상태·포지션·연락 링크를 입력해 팀 카드를 생성합니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| "팀 모집글 생성" 버튼 클릭; 폼 입력 (팀명·소개·모집중여부·포지션·연락링크); "저장" 버튼 클릭 | 생성 폼 표시 → 필드값 검증(팀명·소개 필수) → localStorage `teams` 저장 | 팀 카드 즉시 목록에 추가 (모집 중 배지 포함); (옵션) 수정·모집마감 처리 버튼 제공 |

**팀 모집글 생성 폼 필드**

| 필드 | 키 | 필수 여부 | 설명 |
| ---- | -- | --------- | ---- |
| 팀명 | `name` | 필수 | 팀 이름 |
| 소개 | `intro` | 필수 | 팀 소개 한 줄 이상 |
| 모집 중 여부 | `isOpen` | 필수 | 모집 중(true) / 마감(false) |
| 모집 포지션 | `lookingFor[]` | 선택 | Frontend / Backend / Designer 등 복수 선택 |
| 연락 링크 | `contact.url` | 선택 | 오픈카톡 링크 또는 구글폼 링크 등 |

---

## 4-5. 글로벌 랭킹 (/rankings)

전체 해커톤 참여 기록을 합산한 글로벌 순위를 보여주며, 기간 필터로 범위를 좁힐 수 있습니다.

| 입력 | 동작 | 결과 |
|------|------|------|
| 페이지 진입; (옵션) 기간 필터 선택 (최근 7일·30일·전체) | 전체 `leaderboards`에서 팀별 points 합산, 랭크 정렬; 필터 있으면 해당 기간 데이터만 집계 | rank·닉네임(팀명)·points 테이블 표시 |

# 5. 주요 사용 흐름

정상적인 해커톤 참가 흐름과 각 단계에서 발생할 수 있는 주요 예외 처리 상황을 함께 정리했습니다.

```text
1. 메인 (/) 진입
   ├─ [예외] 초기화/localStorage 오류 → 에러 메시지 + 새로고침 안내 표시
   └─ "해커톤 보기" 카드 클릭

2. 해커톤 목록 (/hackathons) 로드
   ├─ [예외] 데이터 로딩 지연 → 화면에 스피너(로딩 인디케이터) 표시
   └─ "긴급 인수인계 해커톤" 카드 클릭

3. 해커톤 상세 (/hackathons/daker-handover-2026-03) 진입
   ├─ [예외] 해당 slug의 해커톤이 존재하지 않음 → 안내 메시지 + "목록으로 돌아가기" 표시
   │
   ├─ [팀] 탭 확인
   │   ├─ [예외] 등록된 팀이 하나도 없음 → "아직 팀이 없습니다. 팀을 만들어보세요!" + 생성 버튼 표시
   │   └─ "이 해커톤 팀 구성" 클릭 → /camp?hackathon=daker-handover-2026-03 이동
   │       └─ 팀 모집글 생성 폼 작성 및 제출
   │
   ├─ 페이지 복귀 후 [제출] 탭 확인
   │   ├─ 제출 가이드 확인 후 폼 작성 (메모, 파일 URL 등 입력)
   │   ├─ [예외] 필수 항목 미입력 → 인라인 에러 메시지 노출, "제출" 버튼 비활성화
   │   └─ "제출" 클릭 → 정상 저장 및 제출 완료 처리
   │
   └─ [리더보드] 탭 확인
       ├─ 점수 산식에 따른 전체 순위 테이블 렌더링
       └─ [예외] 제출 내역이 없는 팀 → 팀명 옆에 "미제출" 배지 표시, 순위 부여 제외

4. 글로벌 랭킹 (/rankings) 진입
   └─ 기간 필터 적용 및 전체 해커톤 통합 누적 포인트 순위 확인
```

# 6. 개발 및 개선 계획

## Phase 1 — 일반 기능 구현 (Day 1 ~ Day 4)

필수적인 핵심 참가 흐름을 우선 완성하는 기간입니다.

| 일정 | 개발 목표 및 상세 내용 |
| ---- | ---------------------- |
| Day 1 | **프로젝트 세팅 및 공통 UI**: Vite+React 초기 설정, 라우팅, 전역 상태 및 localStorage 시딩 로직 구현. 메인 페이지(`/`) 제작. |
| Day 2 | **해커톤 조회 기능**: 해커톤 목록(`/hackathons`) 페이지 렌더링. 상세(`/hackathons/:slug`) 페이지의 7개 탭 중 정적 데이터 기반(개요·평가·일정·상금 등) 뷰 구현. |
| Day 3 | **팀원 모집 기능**: 팀원 모집(`/camp`) 페이지 전체 및 해커톤 지정 필터 구현. 팀 생성 폼 개발 및 데이터 연동. 상세 페이지의 팀 탭 연결 로직 완성. |
| Day 4 | **제출 및 랭킹**: 제출(Submit) 폼 검증 로직 개발 및 localStorage 연동. 리더보드 기능 구현 및 점수 산식 연계, 글로벌 랭킹(`/rankings`) 페이지 제작. |

## Phase 2 — 옵션 기능 및 고도화 (Day 5 ~ Day 6)

UX 개선과 편의 옵션을 추가하여 서비스의 완성도를 높이는 기간입니다.

| 일정 | 개발 목표 및 상세 내용 |
| ---- | ---------------------- |
| Day 5 | **필터 및 추가 인터랙션**: 해커톤 목록의 상태/태그 필터, 글로벌 랭킹의 기간별 필터 구현. 팀 탭에서 쓸 수 있는 팀 초대/수락/거절 버튼 로직 개발. |
| Day 6 | **팀 모집 관련 고도화**: 팀 모집글 수정 및 마감(`isOpen` 전환) 기능 개발. 연락처(오픈카톡/구글폼) 연동 강화, 플랫폼 내 채팅/쪽지 연결(기능이 있다면) 및 전체 UI/UX 폴리싱. |

## Phase 3 — 버그 픽스 및 배포 (Day 7)
