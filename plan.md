# 1. 서비스 개요

- 구현할 서비스가 무엇인지 한 눈에 보이도록 서비스명, 한줄 소개, 개발 목표를 정리합니다.
- 누가 사용하는 서비스인지(타겟 사용자)와 사용자에게 제공되는 가치(기대효과)를 함께 작성합니다.

# 2. 페이지 구성

- 서비스를 이루는 주요 페이지와 페이지 간 연결관계를 정리합니다.
- 각 페이지에서 무엇을 보여주고 어떤 입력/버튼으로 어떤 행동이 가능한지 1~2줄로 설명합니다.

# 3. 시스템 구성

- 서비스가 어떤 구조로 동작하는지를 간단히 설명합니다.

# 4. 핵심 기능 명세

- 구현할 기능을 정리하되, 각 기능은 "입력 -> 동작 -> 결과"과 보일 수 있도록 작성합니다.
- (직접 작성한 목록, 화면 설명용 문구, 페이지 항목 구성 등)

# 5. 주요 사용 흐름

- 사용자가 서비스를 이용하는 과정을 단계별로 나누어 작성합니다.
- 정보 없음/입력 오류/접속 불가 같은 상황에서 화면이 어떻게 처리되는지도 짧게 포함합니다.

# 6. 개발 및 개선 계획

- 기본 구현을 완성한 뒤 추가할 기능/UX 개선 아이디를 제안합니다.
- 우선순위/일정 등을 정리합니다.
