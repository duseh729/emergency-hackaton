# 기획서 초안 — 긴급 인수인계 해커톤 플랫폼

---

# 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| **서비스명** | Daker — 긴급 인수인계 해커톤 플랫폼 |
| **한줄 소개** | 기능 명세서만 남기고 사라진 개발자의 문서를 기반으로, 바이브 코딩으로 웹서비스를 구현·배포하는 해커톤 운영 플랫폼 |
| **개발 목표** | 해커톤 목록 조회 → 팀 구성(/camp) → 제출(Submit) → 리더보드까지 이어지는 전체 참가 흐름을 localStorage 기반으로 구현하고 Vercel에 배포한다 |

**타겟 사용자**
- 해커톤에 참가하거나 관심 있는 개발자/디자이너/기획자
- 팀원을 구하고 싶은 솔로 참가자

**기대효과**
- 해커톤 참가의 진입장벽 낮춤 (팀 구성 ~ 제출까지 한 플랫폼에서 해결)
- 공개 리더보드로 경쟁 동기 부여 및 투명한 평가 가시화

# 2. 페이지 구성

```
메인페이지 (/)
  ├── 해커톤 목록 (/hackathons)
  │     └── 해커톤 상세 (/hackathons/:slug)
  │           ├── 개요(Overview) / 안내(Info)
  │           ├── 평가(Eval)
  │           ├── 일정(Schedule)
  │           ├── 상금(Prize)
  │           ├── 팀(Teams)  ←→  팀원 모집 (/camp)
  │           ├── 제출(Submit)
  │           └── 리더보드(Leaderboard)
  └── 랭킹 (/rankings)
```

| 페이지 | URL | 주요 내용 | 주요 액션 |
|--------|-----|-----------|-----------|
| 메인 | `/` | 서비스 소개 + 빠른 진입 카드 3개 | 해커톤 보기 / 팀 찾기 / 랭킹 보기 버튼 클릭 → 각 페이지 이동 |
| 해커톤 목록 | `/hackathons` | 해커톤 카드 리스트 (제목·상태·태그·기간·참가자수 표시) | 카드 클릭 → `/hackathons/:slug` 이동, (옵션) 상태·태그 필터 |
| 해커톤 상세 | `/hackathons/:slug` | 탭 구조로 7개 섹션 표시 | 탭 전환, 팀 구성 초대, 제출 폼 입력·저장, 리더보드 확인 |
| 팀원 모집 | `/camp` | 전체 팀 리스트 (쿼리 필터: `?hackathon=slug`) | 팀 카드 보기, 팀 모집글 생성 폼 제출 |
| 랭킹 | `/rankings` | 글로벌 랭킹 테이블 (rank/닉네임/points) | (옵션) 기간 필터: 최근 7일 / 30일 / 전체 |

**공통 UI**
- 상단 네비게이션 바: 메인 / `/hackathons` / `/camp` / `/rankings` 항목 고정 표시
- 로딩 스피너, 데이터 없음 안내, 에러 메시지 3종 상태 처리

# 3. 시스템 구성

```
[브라우저]
  ├── React (Vite) SPA
  │     ├── React Router — 클라이언트 사이드 라우팅
  │     └── 컴포넌트 레이어
  │           ├── Pages (/, /hackathons, /hackathons/:slug, /camp, /rankings)
  │           └── UI Components (NavBar, HackathonCard, TeamCard, LeaderboardTable …)
  │
  └── localStorage (영속 데이터 저장소)
        ├── hackathons   — 해커톤 목록·상세 (초기값: 예시 JSON 시딩)
        ├── teams(camp)  — 팀 모집글
        ├── submissions  — 제출 내역 (plan/web/pdf)
        └── leaderboards — 리더보드 점수

[배포] Vercel (정적 SPA 배포, 외부 접속 가능)
[외부 API/DB 없음] — 심사자가 별도 키 없이 전체 기능 확인 가능
```

**데이터 초기화 전략**
- 앱 최초 실행 시 `예시자료/*.json`을 localStorage에 시딩
- 이후 CRUD는 localStorage에서 직접 처리 (새로고침 유지)

# 4. 핵심 기능 명세

## 4-1. 해커톤 목록 조회

| 구분 | 내용 |
|------|------|
| 입력 | 페이지 진입, (옵션) 상태 필터 선택(진행중/종료/예정), 태그 필터 선택 |
| 동작 | localStorage `hackathons` 읽어 카드 렌더링; 필터 조건 일치 항목만 표시 |
| 결과 | 해커톤 카드 리스트 표시 — 제목·상태 배지·태그·시작~종료일; 카드 클릭 시 상세 이동 |

## 4-2. 해커톤 상세 (7개 탭 섹션)

| 탭 | 입력 | 동작 | 결과 |
|----|------|------|------|
| 개요/안내 | 탭 클릭 | slug에 맞는 detail 데이터 로드 | 대회 요약·팀 정책·공지·규정 링크 표시 |
| 평가 | 탭 클릭 | eval 섹션 렌더링 | 평가 지표명·설명·가중치(참가자 30%/심사위원 70%) 표시 |
| 일정 | 탭 클릭 | schedule 섹션 렌더링 | 마일스톤 타임라인 표 표시 |
| 상금 | 탭 클릭 | prize 섹션 렌더링 | 1~3위 상금 표시 |
| 팀 | 탭 클릭 | `/camp?hackathon=slug` 쿼리로 팀 목록 로드 | 해당 해커톤 팀 카드 표시 + "이 해커톤 팀 구성" 버튼 |
| 제출 | 제출 폼 입력·버튼 클릭 | 제출 가이드 표시 → 폼(메모/파일URL) 입력 → localStorage `submissions` 저장 | 제출 완료 알림; 리더보드 "미제출" → "제출 완료"로 반영 |
| 리더보드 | 탭 클릭 | `leaderboards[slug]` 읽기 + 제출 내역과 join | 순위 테이블(rank·팀명·score·제출일), 미제출 팀은 순위 없음 표시 |

## 4-3. 팀원 모집 (/camp)

| 구분 | 내용 |
|------|------|
| 입력 | 페이지 진입 또는 `/camp?hackathon=slug` 쿼리 진입; "팀 모집글 생성" 버튼 클릭 |
| 동작 | 쿼리 있으면 해당 해커톤 팀만 필터; 생성 폼 — 팀명(필수)/소개(필수)/모집중 여부/모집 포지션/연락 링크 입력 → localStorage `teams` 저장 |
| 결과 | 팀 카드 목록 갱신; 생성된 팀이 즉시 카드로 표시 |

## 4-4. 글로벌 랭킹 (/rankings)

| 구분 | 내용 |
|------|------|
| 입력 | 페이지 진입; (옵션) 기간 필터 선택 |
| 동작 | 전체 `leaderboards`에서 팀별 points 합산, 랭크 정렬 |
| 결과 | rank / 닉네임(팀명) / points 테이블 표시 |

# 5. 주요 사용 흐름

## 정상 흐름 (팀 구성 → 제출 → 리더보드 확인)

```
1. 메인(/) 진입
   └─ "해커톤 보기" 카드 클릭

2. 해커톤 목록(/hackathons) 확인
   └─ "긴급 인수인계 해커톤" 카드 클릭

3. 해커톤 상세(/hackathons/daker-handover-2026-03) 진입
   ├─ [팀] 탭 → "이 해커톤 팀 구성" 클릭 → /camp?hackathon=daker-handover-2026-03
   │      └─ 팀 모집글 생성 폼 작성 (팀명·소개·포지션·연락처) → 저장
   ├─ 상세 페이지 복귀 → [제출] 탭 → 가이드 확인 → 제출 폼 입력(기획서/웹링크/PDF) → "제출" 클릭
   └─ [리더보드] 탭 → 본인 팀 순위 확인

4. 랭킹(/rankings) → 전체 해커톤 통합 순위 확인
```

## 예외 처리

| 상황 | 화면 처리 |
|------|-----------|
| 데이터 로딩 중 | 스피너(로딩 인디케이터) 표시 |
| 해당 slug 해커톤 없음 | "해커톤을 찾을 수 없습니다" 안내 + 목록으로 돌아가기 버튼 |
| 팀 목록 비어있음 | "아직 팀이 없습니다. 팀을 만들어보세요!" + 생성 버튼 |
| 제출 폼 필수 항목 미입력 | 인라인 에러 메시지 표시, 제출 버튼 비활성 |
| 제출 내역 없는 팀의 리더보드 | "미제출" 배지 표시, 순위 제외 |
| 네트워크/localStorage 오류 | 에러 메시지 + 새로고침 유도 안내 |
